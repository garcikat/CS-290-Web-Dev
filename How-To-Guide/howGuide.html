How the API Works

What is an API

An Application Program Interace is way for software to talk to one another within a structured format. For example, let's say an app wants to the top 10 headlines from a news webpage. Without an API, it would have to read the webapge and interpret the content. If the news webpage would happen to change (something that happens often), then the app would not likely find the information it is looking for. But with and API, an app can send a request to the news webapge in a certain format and the news webpage responds back with a structured response. The app can access the data from the webpage much more easily regardless how much the webpage changes.

req.open("GET", "http://www.anapioficeandfire.com/api/characters", false);

event.preventDefault(); <--common problems

Let's break it down.

var req = new XMLHttpRequest();  - This creates an XMLHttpRequest which is responsible for creating the HTTP request and receiving the returned data. It's the beginning of the conversation between software.

req.open("GET", "http://www.anapioficeandfire.com/api/characters", true); - The request object is opening a GET request from the webpage http://www.anapioficeandfire.com/api/characters. The "GET" argument is the type of request are sending. Its basically used for "getting" data. The last argument "true" means that this is a synchronous request.

var response = JSON.parse(req.responseText);) - Here we are taking the response text sent to us by http://www.anapioficeandfire.com/api/characters and parsing it into a readable format.
